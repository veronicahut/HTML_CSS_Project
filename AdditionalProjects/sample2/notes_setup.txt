Make PHP available to VS Code (two options)

Option A — Add to Windows PATH (recommended so terminals and other apps see PHP)

GUI (safe):
Open Start → type "Environment Variables" → "Edit the system environment variables".
Click "Environment Variables...".
Under "System variables" select "Path" (from listed variables) → Edit → New → add the directory (e.g., C:\xampp\php).

OK, restart VS Code.

PowerShell (quick, modifies persistent user PATH; be careful with long PATHs):
# replace with your php folder (without php.exe)
setx PATH "$env:Path;C:\xampp\php"

After setx you must restart your PowerShell/VS Code windows to pick up the change.

In powershell, click trash icon

Option B — Set the VS Code setting php.executables (workspace or user)

Open Command Palette (Ctrl+Shift+P) → "Preferences: Open Settings (JSON)".

  "php": "C:\\xampp\\php\\php.exe"
}

Save and restart VS Code.

Completed: Explain UAC impact (1/5)

Short answer

No — the UAC warning is usually not a serious problem for local development, but it explains that some 
actions (writing to protected folders, registering services, binding protected ports) may require 
administrative rights. Follow the recommended steps below to avoid issues.
What the warning means

UAC restriction: Windows User Account Control prevents programs from performing privileged actions 
(installing services, writing into C:\Program Files, binding to system-protected locations) unless run 
with elevated rights.
XAMPP impact: XAMPP may be limited if installed into a protected folder or if you don’t run the 
Control Panel as an administrator. Typical symptoms: Apache/MySQL fail to start, can't install services, 
or permission errors when writing web files.
Recommended actions

Install/run as Administrator: Right-click the XAMPP installer and choose Run as administrator. 
If you already installed without admin rights, uninstall and reinstall using admin run.
Install location: Choose C:\xampp (root of the drive), not C:\Program Files. Installing to 
C:\xampp avoids many UAC write restrictions.
Run Control Panel as admin: After install, right-click xampp-control.exe → Run as administrator to 
start services reliably and to install them as Windows services (use the Svc checkbox for Apache/MySQL).
Don’t disable UAC globally: Not recommended — better to run only the necessary apps as admin.
If you already installed XAMPP without admin and don't want to reinstall

Always run the XAMPP Control Panel as administrator when starting Apache/MySQL.
If you placed files in a protected folder, move the project to a non-protected path (for example, 
C:\xampp\htdocs\your_project).

Next: make PHP available to VS Code

After running this, close and reopen PowerShell and VS Code to pick up the change.
